# Employee Portal Kickoff — 2026-06-15

## 1. Objective
Kick off the Acme Employee Portal project: agree on the goal, the authentication approach, and immediate next steps.

## 2. Agenda
1. Project goal & scope
2. Authentication approach
3. Timeline & next steps

## 3. Discussion

### 3.1 Project goal & scope
**Alice:** "We want one place where employees handle HR tasks — payslips, leave, profile. No more emailing HR for every payslip. For the first phase we'll keep it web-only; mobile native is later."

**Ben:** "Makes sense. Do we need manager approvals in phase one?"

**Alice:** "No — approvals are out of scope for now. We just want self-service view and submit."

### 3.2 Authentication approach
**Ben:** "For login, we should use corporate SSO. We already standardize on Okta as the identity provider. I don't want a separate portal password."

**Alice:** "Agreed. Let's go with Okta. SSO needs to be live by July so we can start onboarding pilot users."

**Ben:** "Okta integration lives in an Identity module. Alice, can you confirm the IdP licenses by next Friday — I need to know how many seats."

**Alice:** "Yes, I'll confirm the license count by the 27th."

**Dana:** "One thing we haven't decided: do contractors get portal access? They're not full employees but some need payslip-equivalent docs."

**Alice:** "Good question — I don't know yet. Let's flag that and I'll check with HR policy."

### 3.3 Timeline & next steps
**Ben:** "If SSO is live by July, we can pilot in late July and target full go-live August first."

**Alice:** "Sounds right. Action for me: confirm licenses. Action for Ben: draft the Identity module design. Let's sync next week."

## 4. Raw decisions / actions mentioned
- Okta for SSO.
- Alice confirms IdP licenses by 2026-06-27.
- Ben drafts Identity module design.
- Contractors access TBD.
